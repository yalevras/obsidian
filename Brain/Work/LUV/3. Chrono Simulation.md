fmu_wrapper.cpp needs indexes for VR values

**fmu_wrapper.cpp**

1. **Single-wheel assumption** — `FMUInstance` holds one `WheelProp` and the VR table maps to it directly. Now that `SimModel` holds a vector of `WheelModel`s, the wrapper has no way to set inputs per-wheel or get outputs per-wheel. For now this only works for `NUM_WHEELS = 1`. If multi-wheel is a near-term goal, you'll need indexed VRs (e.g. `VR_POS_X_0`, `VR_POS_X_1`) or a wheel-index parameter set before the kinematic VRs.
2. **`DoStep` is never called** — the wrapper implements `fmi2SetReal`/`fmi2GetReal` but there's no `fmi2DoStep`. Without it the FMU is not a valid Co-Simulation FMU and Simulink won't be able to advance time. You need something like:

cpp

```cpp
    fmi2Status fmi2DoStep(fmi2Component c, fmi2Real currentCommunicationPoint,
                          fmi2Real communicationStepSize, fmi2Boolean noSetFMUStatePriorToCurrentPoint) {
        FMUInstance* inst = (FMUInstance*)c;
        inst->model->SetInputs(inst->pos, inst->vel, inst->omega, inst->quat);
        inst->model->DoStep(communicationStepSize);
        inst->model->GetOutputs(inst->force, inst->torque);
        inst->time += communicationStepSize;
        return fmi2OK;
    }
```

3. **`SetWheelProperties` is called on `WheelModel` but that method doesn't exist on `SimModel`** — `fmi2ExitInitializationMode` calls `inst->model->SetWheelProperties(...)` but `model` is a `WheelModel`, not a `SimModel`. Check that the method signature matches; `WheelModel::SetWheelProperties` takes a `WheelProp&` not a `shared_ptr`.
4. **`fmi2Instantiate` null check is after the dereference risk** — `if (!inst)` comes after `inst->initialized = false`. Swap the order.
5. **Missing required no-op stubs** — FMI 2.0 requires these to be exported even if unused: `fmi2FreeInstance`, `fmi2Terminate`, `fmi2Reset`, `fmi2GetInteger`, `fmi2SetInteger`, `fmi2GetBoolean`, `fmi2SetBoolean`. Without them the FMU will fail to load in most environments.
